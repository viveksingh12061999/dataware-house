# warehouse.fact_sales

## Description

The **fact_sales** table is the central transactional fact table in the Retail Data Warehouse.

Each record represents **one product sold in one customer order**.

The table stores measurable business metrics such as quantity, revenue, cost, discount, and profit, while linking to descriptive dimension tables through surrogate keys.

---

## Grain

**One row = One Product sold in One Order**

Example:

| Order ID | Product | Quantity |
|----------|----------|----------|
|1001|Laptop|1|
|1001|Mouse|2|

This results in **two rows** in the fact table.

| Column | Data Type | Nullable | Default | Constraints | Description |

| cost | NUMERIC(12,2) | Yes | NULL | — | Total product cost |
| profit | NUMERIC(12,2) | Yes | NULL | — | Profit earned (Revenue − Cost) |

## Measures

The following columns are used for business analytics:

- quantity
- revenue
- cost
- profit
- discount

These columns are aggregated in reports and dashboards.

---

## Business Rules

- Quantity must always be greater than zero.
- Unit price cannot be negative.
- Discount cannot be negative.
- Every transaction must reference valid dimension records.
- Every sales record must belong to a customer, product, region, and date.
